## 📝 핵심 개념 정리

### 1. JavaScript란?

**한 줄 정의**: 원래 "웹페이지에 생명을 불어넣기 위해" 만들어진 프로그래밍 언어.

- **탄생 배경**: 1995년, 초기 웹은 글자와 그림만 있는 **정적인 문서**였음. 여기에 "버튼을 누르면 반응하고, 입력을 검사하는" 동작을 넣으려고 브라우저 안에서 도는 언어로 등장. → 그래서 처음엔 오직 브라우저에서만 실행됐음.

- **지금은 어디서든 실행**: **자바스크립트 엔진**이라는 프로그램만 있으면 됨.
  | 실행 환경 | 엔진 | 용도 |
  |---|---|---|
  | Chrome, Edge, Opera | **V8** | 웹페이지 |
  | Firefox | SpiderMonkey | 웹페이지 |
  | 서버, PC 프로그램 | **Node.js** (V8 기반) | 백엔드, 도구, 앱 |

  → 즉 요즘 JS는 **프론트엔드(브라우저) + 백엔드(서버)** 를 하나의 언어로 다룰 수 있음. 이게 JS의 가장 큰 강점.

- **엔진은 내 코드를 어떻게 실행할까?** (대략의 흐름)
  1. **파싱(parsing)**: 소스코드를 읽어서 문법 구조로 분석
  2. **컴파일**: 분석한 걸 기계어(컴퓨터가 바로 아는 언어)로 변환
  3. **실행**: 기계어를 돌림. 실행하는 동안에도 계속 **최적화**를 함
  > 옛날엔 JS가 "느린 인터프리터 언어"였지만, 지금은 이 컴파일 최적화 덕분에 매우 빠름.

- **이름 주의**: `JavaScript`에 'Java'가 들어가지만 **Java와 전혀 다른 언어**임. 1995년 당시 Java가 인기라서 마케팅용으로 붙인 이름일 뿐. 헷갈리지 말 것.

### 2. 브라우저에서 JavaScript가 할 수 있는 것 / 없는 것

브라우저 속 JS는 "웹페이지를 다루는 일"에 특화돼 있음.

**✅ 할 수 있는 것**
- 페이지의 HTML을 바꾸기, 새 태그 추가/삭제, CSS 스타일 변경
- 마우스 클릭·키보드 입력 등 사용자 행동에 반응
- 서버에 요청 보내고 응답 받기 (`fetch`, AJAX) — 페이지 새로고침 없이 데이터 주고받기
- 쿠키·`localStorage`에 데이터 저장/읽기
- 팝업 띄우기, 소리 재생 등

**🚫 (보안 때문에) 할 수 없는 것**
- 사용자 허락 없이 **내 컴퓨터의 아무 파일이나 읽고 쓰기** ❌
- 다른 탭/사이트의 정보를 몰래 훔쳐보기 ❌ (**동일 출처 정책 Same-Origin Policy**: `naver.com`의 스크립트가 `google.com`의 내용을 함부로 못 봄)
- OS 시스템 기능 마음대로 호출 ❌

> **왜 막아놨을까?** 우리는 매일 아무 웹사이트나 방문함. 만약 그 사이트의 JS가 내 하드디스크를 읽을 수 있다면 큰일. 그래서 브라우저 JS는 일부러 **가둬둔(sandbox)** 상태로 안전하게 동작함. (반면 Node.js는 이런 제약 없이 파일 접근 가능 — 대신 내가 직접 실행하는 신뢰된 코드니까.)

### 3. JavaScript "위에 얹은" 언어들 (트랜스파일)

- 브라우저는 결국 **JS만** 이해함. 그래서 다른 문법을 쓰고 싶으면 그 코드를 **JS로 자동 변환(트랜스파일 transpile)** 해서 사용.
- 대표적인 것들:
  - **TypeScript** (Microsoft): JS에 **타입(자료형 명시)** 을 추가 → 실수를 미리 잡아줌. 요즘 실무 표준급. *(나중에 여유되면 꼭 배우면 좋음)*
  - CoffeeScript, Dart, Brython 등
- 핵심: 이런 언어를 배우더라도 **결국 JS로 바뀌므로, JS 기초를 아는 게 먼저**임.

### 4. 매뉴얼과 명세서 — 궁금할 때 어디서 찾나

| 자료 | 성격 | 언제 쓰나 |
|---|---|---|
| **MDN** ([developer.mozilla.org](https://developer.mozilla.org)) | 함수·메서드 사용법 + 예제 | **평소에 제일 자주** 씀. 구글에 `MDN 배열 map` 처럼 검색 |
| **ECMA-262 명세서** ([tc39.es/ecma262](https://tc39.es/ecma262/)) | 언어의 공식 정의(정답) | 아주 정확한 동작이 궁금할 때. 딱딱함 |
| **Can I use** ([caniuse.com](https://caniuse.com)) | 브라우저 지원 여부표 | "이 기능 사파리에서 되나?" 확인할 때 |

> JS는 **매년 새 버전**이 나옴(ES2015, ES2016… 매해 조금씩 기능 추가). 그래서 "완벽히 다 외우기"보다 **필요할 때 MDN에서 찾는 습관**이 훨씬 중요함.

### 5. 코드 에디터

코드를 작성하는 프로그램. 크게 두 종류:

- **IDE (통합 개발 환경)**: 프로젝트 전체를 관리, 자동완성·디버깅·git 등 기능 풍부.
  - 👉 **VS Code** (무료, 가장 대중적, 추천), WebStorm(유료, 강력)
- **경량 에디터**: 파일 하나 빠르게 열 때. Sublime Text, Notepad++ 등.

> **결론: VS Code 설치하고 시작하면 됨.** 처음엔 굳이 다른 거 고민하지 말 것.

---

### 6. 개발자 콘솔 (개발자 도구) ⭐ 오늘의 핵심 실습

브라우저에 내장된, 개발자를 위한 도구. **오류 확인**과 **코드 직접 실행**을 여기서 함.

- **여는 법**
  | OS | 단축키 |
  |---|---|
  | Windows / Linux | `F12`  또는  `Ctrl + Shift + J` (콘솔 바로) |
  | Mac | `Cmd + Option + J` |

- **Console 탭** 사용법
  - 코드 한 줄 입력 후 `Enter` → 즉시 실행됨
  - 여러 줄을 쓰고 싶으면 줄 끝에서 `Shift + Enter`로 줄바꿈, 마지막에 `Enter`로 실행
  - 페이지에 에러가 나면 여기에 **빨간 글씨**로 표시됨 → 개발할 때 항상 열어두는 습관!

## 💻 코드 실습
```javascript
// 브라우저에서 F12 → Console 탭에 아래를 한 줄씩 입력하고 Enter 해보세요

console.log("Hello, JavaScript!");   // 콘솔에 메시지 출력 (개발할 때 가장 많이 쓰는 명령)

alert("첫 자바스크립트 실행!");        // 화면에 팝업 경고창 띄우기

1 + 2                                // 계산기처럼도 사용 가능 → 결과: 3

"자바" + "스크립트"                    // 문자열도 더할 수 있음 → "자바스크립트"
```
