## 📝 핵심 개념 정리

### 1. 용어 정리

- **피연산자(operand)**: 연산자가 연산을 수행하는 대상. `5 * 2`에서 `5`와 `2`가 피연산자다.
- **단항(unary) 연산자**: 피연산자를 하나만 받는 연산자.
- **이항(binary) 연산자**: 피연산자를 두 개 받는 연산자.

```javascript
let x = 1;
x = -x;          // 단항 마이너스, 부호를 뒤집는다 → -1

let y = 5 - 2;   // 이항 마이너스, 두 값을 뺀다 → 3
```

같은 `-` 기호라도 피연산자 개수에 따라 하는 일이 다르다.

### 2. 산술 연산자

덧셈 `+`, 뺄셈 `-`, 곱셈 `*`, 나눗셈 `/`에 더해 두 가지가 더 있다.

```javascript
console.log(5 % 2);  // 1, 나머지(remainder)
console.log(8 % 4);  // 0

console.log(2 ** 3);   // 8, 거듭제곱 2*2*2
console.log(4 ** 0.5); // 2, 제곱근도 거듭제곱으로 표현할 수 있다
```

- `%`는 "퍼센트"와 관련이 없고, **나눈 뒤의 나머지**를 반환한다.
- `**`는 `a`를 `b`번 곱한 값을 반환하며, 지수가 소수여도 동작한다.

### 3. 이항 `+` — 문자열 연결

피연산자 중 **하나라도 문자열이면** 나머지도 문자열로 변환한 뒤 이어 붙인다.

```javascript
console.log("my" + "string"); // "mystring"
console.log("1" + 2);         // "12"
console.log(2 + "1");         // "21"

console.log(2 + 2 + "1");     // "41" (왼쪽부터 계산: 2+2=4, 그 뒤 "1"과 연결)
console.log("1" + 2 + 2);     // "122"
```

> ⚠️ 이런 동작은 이항 `+`에만 해당한다. 다른 산술 연산자는 피연산자를 숫자로 변환한다.

```javascript
console.log(6 - "2");   // 4
console.log("6" / "2"); // 3
```

### 4. 단항 `+` — 숫자형으로 변환

숫자에 붙이면 아무 일도 하지 않지만, 숫자가 아닌 값에 붙으면 숫자형으로 변환한다.
`Number(value)`와 동일하게 동작하는 짧은 문법이다.

```javascript
let x = 1;
console.log(+x); // 1

console.log(+true); // 1
console.log(+"");   // 0

// 폼에서 받은 값은 문자열이므로 그대로 더하면 연결된다
let apples = "2";
let oranges = "3";

console.log(apples + oranges);   // "23" (문자열 연결)
// 단항 덧셈(14), 이항 덧셈(11)
console.log(+apples + +oranges); // 5   (각각 숫자로 변환한 뒤 덧셈)
```

단항 연산자는 이항 연산자보다 우선순위가 높기 때문에 `+apples`가 먼저 계산된다.

### 5. 연산자 우선순위(operator precedence)

우선순위 숫자가 클수록 먼저 실행되고, 같으면 왼쪽에서 오른쪽 순으로 계산된다.

| 우선순위 | 연산자 | 기호 |
|---:|---|---|
| 14 | 단항 플러스 | `+` |
| 14 | 단항 마이너스 | `-` |
| 13 | 거듭제곱 | `**` |
| 12 | 곱셈 | `*` |
| 12 | 나눗셈 | `/` |
| 11 | 덧셈 | `+` |
| 11 | 뺄셈 | `-` |
| 2 | 할당 | `=` |

단항 `+`(14)가 이항 `+`(11)보다 우선순위가 높고, 할당 `=`은 거의 마지막에 실행된다.

### 6. 할당(assignment) 연산자 `=`

`=`도 우선순위가 아주 낮은 **연산자**이며, 할당한 값을 그대로 반환한다.

```javascript
let a = 1;
let b = 2;

let c = 3 - (a = b + 1);
console.log(a); // 3
console.log(c); // 0

// 할당 연산자는 오른쪽부터 실행되며 체이닝이 가능하다
let x, y, z;
x = y = z = 2 + 2;
console.log(x, y, z); // 4 4 4
```

> 💡 동작은 하지만 가독성이 떨어지므로 위와 같은 스타일은 권장되지 않는다.

### 7. 복합 할당 연산자

`*=`, `/=`, `+=`, `-=` 처럼 연산과 할당을 한 번에 처리한다.
우선순위는 일반 할당과 같아서 **오른쪽 표현식을 모두 계산한 뒤** 적용된다.

```javascript
let n = 2;
n += 5; // n = n + 5 → 7
n *= 2; // n = n * 2 → 14

let m = 2;
m *= 3 + 5; // 오른쪽이 먼저 계산되어 m = 2 * 8 → 16
```

### 8. 증가·감소 연산자

`++`는 1 증가, `--`는 1 감소시키며 **변수에만** 사용할 수 있다.
`5++` 같은 코드는 에러가 난다.

```javascript
let counter = 2;
counter++;
console.log(counter); // 3
```

전위형과 후위형은 **반환값**이 다르다.

```javascript
let a = 1;
let b = 1;

console.log(++a); // 2, 먼저 증가시킨 뒤 새로운 값을 반환
console.log(b++); // 1, 이전 값을 반환한 뒤 증가
console.log(b);   // 2
```

- **전위형 `++a`**: 값을 증가시킨 후 **증가한 값**을 반환한다.
- **후위형 `a++`**: **증가 전 값**을 반환한 후 값을 증가시킨다.
- 반환값을 사용하지 않는다면 둘의 차이는 없다.

### 9. 비트 연산자와 쉼표 연산자

비트 연산자는 인수를 32비트 정수로 바꿔 비트 단위로 연산한다.
(`&` AND, `|` OR, `^` XOR, `~` NOT, `<<`, `>>`, `>>>`) 웹 개발에선 쓰임새가 적어 필요할 때 찾아보면 된다.

쉼표 연산자 `,`는 표현식을 여러 개 평가한 뒤 **마지막 결과만** 반환한다.
우선순위가 `=`보다도 낮아 괄호가 필요하다.

```javascript
let a = (1 + 2, 3 + 4);
console.log(a); // 7 (3+4의 결과만 남는다)
```

## 💻 코드 실습
```javascript
// 나머지와 거듭제곱
console.log(5 % 2);   // 1
console.log(2 ** 10); // 1024

// 이항 + 는 문자열이 하나라도 있으면 연결한다
console.log(2 + 2 + "1"); // "41"
console.log("1" + 2 + 2); // "122"

// 다른 산술 연산자는 숫자로 변환한다
console.log("6" - "2"); // 4
console.log("6" / "2"); // 3

// 단항 + 로 숫자 변환
let apples = "2";
let oranges = "3";
console.log(apples + oranges);   // "23"
console.log(+apples + +oranges); // 5

// 복합 할당은 오른쪽을 먼저 계산한다
let n = 2;
n *= 3 + 5;
console.log(n); // 16

// 전위형 vs 후위형
let a = 1, b = 1;
console.log(++a); // 2
console.log(b++); // 1
console.log(b);   // 2

// 쉼표 연산자
console.log((1 + 2, 3 + 4)); // 7
```

### 1. 후위형과 전위형
> 중요도: 5
>
> 아래 코드에서 `a`, `b`, `c`, `d`의 최종 값은 무엇일까요?
> ```javascript
> let a = 1, b = 1;
> let c = ++a;
> let d = b++;
> ```

<details>
<summary>정답 보기</summary>

`a = 2`, `b = 2`, `c = 2`, `d = 1`

- `++a`는 **전위형**이라 `a`를 2로 증가시킨 뒤 **증가한 값 2**를 반환한다 → `c = 2`
- `b++`는 **후위형**이라 **증가 전 값 1**을 반환한 뒤 `b`를 2로 증가시킨다 → `d = 1`
- 두 변수 `a`, `b`는 결국 모두 2가 된다.

</details>

### 2. 할당 연산 결과
> 중요도: 3
>
> 아래 코드 실행 후 `a`와 `x`의 값은 무엇일까요?
> ```javascript
> let a = 2;
> let x = 1 + (a *= 2);
> ```

<details>
<summary>정답 보기</summary>

`a = 4`, `x = 5`

- 괄호 안의 `a *= 2`가 먼저 실행되어 `a`는 4가 된다.
- 복합 할당 연산자도 할당한 값을 **반환**하므로 `a *= 2`의 결과는 4다.
- 따라서 `x = 1 + 4 = 5`가 된다.

</details>

### 3. 형 변환
> 중요도: 5
>
> 아래 표현식의 결과를 예측해보세요.
> ```javascript
> "" + 1 + 0
> "" - 1 + 0
> true + false
> 6 / "3"
> "2" * "3"
> 4 + 5 + "px"
> "$" + 4 + 5
> "4" - 2
> "4px" - 2
> "  -9  " + 5
> "  -9  " - 5
> null + 1
> undefined + 1
> " \t \n" - 2
> ```

<details>
<summary>정답 보기</summary>

```javascript
"" + 1 + 0    // "10"       문자열 연결이 왼쪽부터 이어진다
"" - 1 + 0    // -1         - 는 숫자로 변환하므로 0 - 1 = -1, 이후 +0
true + false  // 1          true → 1, false → 0
6 / "3"       // 2          "3"이 숫자로 변환된다
"2" * "3"     // 6          둘 다 숫자로 변환된다
4 + 5 + "px"  // "9px"      4+5가 먼저 계산된 뒤 문자열과 연결된다
"$" + 4 + 5   // "$45"      왼쪽부터 계속 문자열로 연결된다
"4" - 2       // 2
"4px" - 2     // NaN        "4px"는 숫자로 변환할 수 없다
"  -9  " + 5  // "  -9  5"  이항 + 는 문자열을 연결한다
"  -9  " - 5  // -14        - 는 공백을 무시하고 숫자로 변환한다
null + 1      // 1          Number(null)은 0
undefined + 1 // NaN        Number(undefined)는 NaN
" \t \n" - 2  // -2         공백·탭·개행만 있는 문자열은 0으로 변환된다
```

</details>

## 🔁 복습 메모 (다음날 30초 훑기용)
- 피연산자가 하나면 단항, 둘이면 이항 연산자다. 같은 `-`도 개수에 따라 역할이 다르다.
- `%`는 나머지, `**`는 거듭제곱이며 `4 ** 0.5`처럼 제곱근도 계산할 수 있다.
- 이항 `+`는 피연산자 중 하나라도 문자열이면 **문자열 연결**을 한다.
- 그 외 산술 연산자(`-`, `*`, `/`, `%`)는 피연산자를 항상 숫자로 변환한다.
- 단항 `+`는 `Number(value)`와 같은 역할을 하며 이항 `+`보다 우선순위가 높다.
- `=`도 연산자이며 할당한 값을 반환하고, 우선순위가 낮아 거의 마지막에 실행된다.
- 복합 할당(`*=`, `+=` 등)은 오른쪽 표현식을 모두 계산한 뒤 적용된다.
- 전위형 `++a`는 증가 후의 값을, 후위형 `a++`는 증가 전의 값을 반환한다.
- 쉼표 연산자는 여러 표현식을 평가하고 마지막 결과만 반환한다.
